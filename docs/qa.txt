1. Нужна ли возможность редактирования/удаления исторических данных (оценок, проблем, решений за прошлые периоды), или вся история иммутабельна и редактируются только текущие значения?
История должна быть иммутабельной с версионированием изменений (позже), чтобы отслеживать динамику и прогресс.

2. Как обрабатывать ситуацию, когда количество проблем в сфере меньше 8 - можно ли добавлять проблемы в течение недели, или список проблем фиксируется при еженедельной оценке?
Список проблем должен быть динамическим - пользователь может добавлять/удалять проблемы в любой момент, но до установленного максимума в 8 проблем на сферу.

3. Требуется ли связь между решениями из разных сфер (например, одно решение может улучшить несколько сфер одновременно)?
Решения должны быть привязаны строго к одной проблеме в одной сфере, без кросс-связей между сферами.

4. Как определяется "первый маленький шаг" для решения - это отдельная сущность с собственным статусом выполнения, или это просто текстовое описание внутри решения?
Для начало это просто текстовые описание

5. Должна ли система отправлять напоминания о еженедельной проверке, и если да, то в каком формате (email, push, in-app notification)?
нет

6. Нужна ли визуализация в виде круговой диаграммы (как в классическом "колесе баланса"), или достаточно табличного/списочного представления с числовыми значениями?
Нужна

7. Должна ли система автоматически рассчитывать общий индекс баланса по всем 8 сферам, или это делается только визуально через сравнение оценок?
Только визуально

8. Требуется ли возможность экспорта данных (PDF отчеты, CSV для анализа, Excel для архивирования)?
Пока нет

9. Нужна ли социальная компонента - возможность делиться прогрессом, работать с коучем/наставником, групповая работа?
Пока нет

10. Как часто разрешено обновлять оценку по сфере - только раз в неделю при проверке, или в любой момент по желанию пользователя?
Во время сессии обновляются все. Сессия касается только всех сразу, отдельно каждую менять нельзя.

11. Должна ли система валидировать формулировки проблем и решений на соответствие правилам (проблема без намека на решение, решение с глаголом в начале)?
Может быть позже, пока нет

12. Требуется ли история изменений статуса выполнения решений - когда начали, когда завершили, сколько раз переоткрывали?
Нет

13. Нужна ли возможность архивирования/деактивации решённых проблем, или они просто остаются в списке со статусом "решено"?
Да у нас нет какого-то списка активных проблем, соответственно этот вопрос не актуален

14. Должна ли система предлагать шаблоны типичных проблем и решений для каждой сферы, основанные на best practices?
Нет

15. Требуется ли мультиязычность интерфейса, или достаточно только русского языка?
Русский English, возможность расширения

16. Какая аутентификация предполагается - только email/password, или также OAuth через Google/Apple/Facebook?
только e-mail пассворд, позже OAuth, закладываться на это

17. Нужна ли возможность работы в офлайн-режиме с последующей синхронизацией данных при восстановлении соединения?
Пока нет

18. Должна ли система хранить метаданные о том, в какое время дня пользователь обычно делает оценки и работает с системой, для оптимизации времени напоминаний?
Просто историю с временными отметками

19. Требуется ли API для интеграции с внешними системами (календари, таск-менеджеры типа Todoist/Notion, фитнес-трекеры для сферы "Здоровье")?
Позже, пока нет

20. Какой уровень покрытия тестами требуется для считания этапа завершённым - unit-тесты для бизнес-логики, integration tests для API, e2e тесты для критических пользовательских сценариев?
80+
